\section{Introduction}

I have looked for a bit, and not found a FLOSS implemmentation of remote HTTP
proxy for IPFS instances. Well, here it is (in experimental stages).
IPFS-Remote lets you manage your remote IPFS instances by sending authenticated
requests to the IPFS HTTP API, With IPFS-Remote, you manage a repository of
public RSA keys, the counterparts of which ``users'' (really just key labels)
sign their HTTP requests with. The project is intended to be scalable and
iterable, and provides a minimal set of features so that other developers can
build on top of it if they like. All this repository does is take valid, signed
GET requests and forwards them to the IPFS instance.
